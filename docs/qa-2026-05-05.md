## Bugs identificados durante el QA

### BUG-001 — Login form pedía username (FIXED en esta sesión)

`src/app/login/page.tsx` mostraba dos inputs (Username + Password)
heredados del modelo multi-user. En single-user (post-reset) sólo
hace falta password. **Arreglado**: form simplificado a un único
input password + autoFocus, fetch envía sólo `{ password }`.

### BUG-002 — Pill "Open 2027" del header /tax-ops se superpone con el título

Cuando el viewport es estrecho, la pill "📅 Open 2027" en la esquina
superior derecha del PageHeader de /tax-ops cae encima de la palabra
"Ops" del título h1. Visible en `Tax-Ops` con el quote de Diego
("hay páginas que se abren y se superponen con la anterior").

**Hipótesis**: el `<PageHeader>` no aplica `flex-wrap` o el botón
flotante usa `position: absolute` sin `z-index` claro. Reproducir en
`<1300px` y revisar `src/components/ui/PageHeader.tsx` + el wrapper
en `src/app/tax-ops/page.tsx`.

**Severidad**: media. Interrumpe lectura del título pero no bloquea.

### BUG-003 — Sidebar mantiene `roles` arrays + `filterForRole` (cosmético)

`src/components/app-shell/Sidebar.tsx` aún define `Role` type +
`roles?: ['admin', 'reviewer']` annotations en NavItem/NavGroup +
una función `filterForRole`. En single-user `role` está hardcoded a
`'admin'` (vía `const role: Role = 'admin'`), así que `filterForRole`
es siempre identidad. El código funciona pero pesa 80+ líneas que ya
no significan nada.

**Severidad**: baja. Funcional, sólo es cleanup estético.

## Cómo trabajar la lista

Cada bug que encuentres se convierte en una entrada en `docs/TODO.md`
sección "Esta semana". Cada fix que vaya:

1. Reproduce el bug (1 línea con la ruta + acción).
2. Hipótesis de causa (1 línea).
3. Fix en una rama o directo a main si es trivial.
4. 1 commit con mensaje `fix(area): <qué>`.
5. Verifica build verde antes de push.

Para los más visuales (overflow, alignment), usa preview tools:
- `preview_screenshot` para before/after.
- `preview_inspect` con el CSS property específico para verificar.
